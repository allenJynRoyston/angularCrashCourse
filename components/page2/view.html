Setup

Before jumping into Angular, we need to talk about the controller. Angular can broadly broken down into two main parts: the view and the controller, or in other words, the DOM and the code.



While we won't go into details on how to setup controllers, we will go into details on what you can do IN the controller. If you look into the source code for this tutorial, for example, you'll find that each "page" has it's own view and controller.

The code for the controller looks like this: 

// define a controller			
myApp.controller('page2Controller', function($scope) {
	
	  // code goes here
		
		
});

Breaking it Down:

myApp.controller('page2Controller', function($scope){})

1.) The first part, myApp, is just the name we've decided to call our application. We declare it as soon as possible, typically in the html tag. Mind you it doesn't HAVE to be named myApp (we could call it anything we wanted), but lets keep it simple for this demo.

myApp.controller('page2Controller', function($scope){})

2.) The second part is simply declaring that we're creating a controller, hence the aptly named controller.

myApp.controller('page2Controller', function($scope){})

3.) This is the name of the controller itself. You can name this anything you want JUST so long as it matches what's declared in the view (html).

myApp.controller('page2Controller', function($scope){})

4.) This is where our modules will go. We can talk more about modules later, but for now just recognize modules are what you use to add functionality. There are a wide range of modules you can add out the box ($timeout, $state, $interval, etc), but for right now, understand that $scope will be what we use to place content on to a page. For example... 

	// regular variable
	var name = "Allen" 
	
	// scoped variable
	$scope.name = "Allen"

... are both ways to declare variables, but only scoped varaibles will allow you to interact between the controller and the view.

Keep in mind that you should still use standard variables when doing equations and logic and using $scope to display those results.

You can usually declare up to two thousand (!) scoped variables before it affects browser performance, but if there's no need for them to be on the view, then there's no need to declare them as $scope. Note that you can also declare functions, objects and arrays using $scope.

tl;dr

So in summary, all your logic should be written inside the controller.

			
myApp.controller('page2Controller', function($scope) {
  
  
  
  // scope variables/functions
  $scope.init = function(){
	// do something
  }
	 
  $scope.title = "My Controller";
	 
  $scope.description = {name: "Allen", title: "Code Ninja"};
		
  // standard declaration
  var i = 0; 
  
  function myFunction(){
  	// do something
  }
	
	
		
});
